## 이 온톨로지의 철학
Karpathy의 관점("Software 2.0", "LLM OS", context engineering)을 독서 온톨로지에 적용한다:

1. **엔티티 중심.** 책들이 기술하는 *세계 자체*를 엔티티(Concept·Person·Work·Claim)의 그래프로 모델링한다.
2. **원자적 페이지 + 촘촘한 양방향 링크.** 모든 엔티티는 자기완결적 한 조각이며 `[[...]]`로 연결된다. Obsidian 백링크가 양방향성을 자동 생성한다.
3. **검색·적재 최적화.** 각 파일은 LLM 컨텍스트 창에 그대로 떠넣어도 의미가 통한다. 위키 = LLM의 외장 지식 베이스.
4. **원본 로그 / 증류 위키 분리.** Books/ = append-only 원본 캡처(Human 데이터 불가침). 이 위키 = 거기서 증류된 정제 지식. `ai_*` 프론트매터만이 둘을 잇는 다리다.
5. **독자 레이어 보강.** 세계만 그리면 "이 독자가 무엇에 관심 있나"를 잃으므로 [[profile]]로 독자 관점을 유지한다.

## 핵심 표준 (요약, 상세는 [[methodology]])
- **Work 노드는 Books 노트 자체.** Ontology에 Work 파일을 따로 만들지 않는다(중복 금지).
- **Concept만 개별 파일** → `[[개념명]]` 이 실제 노드로 해석됨.
- 책 프론트매터 `ai_*`는 위키링크(`"[[...]]"`)로 채운다 → 백링크로 양방향 그래프 완성.
- 고아 노드·스텁은 생성하지 않는다(환각 방지).
