# Reading Copilot 클로드 버전 개발

[[2026-05-09]]에서 생각을 디벨롭 했던 것 처럼, 더 이상 앱 형식으로 직접 개발하려고 해서는 안된다. 그건 이미 너무 고루한 방식이 될 것이다. 오늘부터는 클로드와 옵시디언 두가지에 꼭 맞도록 서비스를 개발하려고 한다. 기존 개발된 [[service_def (iOS app)]] 의 형식을 클로드 스킬에 맞게 변경하는 작업이 필요하다.

## 옵시디언 노트 템플릿 설계

Reading Copilot 서비스의 핵심은 나는 대화를 나누거나, 관심있는 자료를 Feeding 했을 뿐인데, Copilot이 알아서 옵시디언 형식의 나의 지식 온톨로지를 만들어 주는 것. 이 지도를 들고다니면서 어떤 모델 어떤 환경에 이식해도, 나의 지식환경을 정확하게 이해할 수 있도록 하고 싶다. 그럴려면 ==개별 옵시디언 노트의 형식(Frontmatter) 설계==가 Reading Copilot의 목적성에 맞도록 잘 되어있어야 한다.

- [x]  `author` `country` 등 약한연결 데이터 표준 정의
- [x] `tag` `cross_themes` 데이터 표준 정의
- [x] 사용자가 인풋으로 넣은 정보와, 에이전트가 분석한 정보를 어떻게 구분할 것인가

##### Front Matter 설정 기준 (Claude)

```
---
# ── LAYER 1: 서지 정보 (AG-BD가 자동 수집) ──
title: "도둑맞은 집중력"
author: "요한 하리"
author_orig: "Johann Hari"
published: 2022
country: "UK"
type: "non-fiction"          # non-fiction | fiction | essay | academic
genre: ["psychology", "social-critique"]

# ── LAYER 2: 독서 상태 (AG-OR/TE가 자동 갱신) ──
status: "reading"            # to-read | reading | done | paused
started: 2026-05-01
finished:
highlights_count: 14

# ── LAYER 3: 온톨로지 (AG-IS가 자동 생성) ──
ontology_nodes: ["주의력", "디지털환경", "인지과학"]
interest_score: 0.87         # AG-IS가 하이라이트 밀도로 산출
connected_books:             # 연결된 책 노트 링크
  - "[[딥 워크]]"
  - "[[스마트폰에서 뇌를 구하라]]"
cross_themes: ["몰입", "멀티태스킹 비용"]
---
```

##### 노트 간 연결 기준 (Claude)

| 연결 강도     | 기준                                | 구현 방식                                       |
| --------- | --------------------------------- | ------------------------------------------- |
| **강한 연결** | 같은 핵심 개념을 공유 (하이라이트 키워드 겹침 3개 이상) | `[[책 제목]]` 양방향 링크 + `connected_books` 프론트매터 |
| **중간 연결** | 같은 온톨로지 노드에 속함 (장르·주제가 같음)        | 공유 `#tag` + `cross_themes` 프론트매터            |
| **약한 연결** | 같은 저자 또는 같은 시대·국가의 지적 흐름          | `author` / `country` / `published` 메타데이터 쿼리 |




## 개별 에이전트를 클로드 스킬로 변환

앱서비스 안에서 뒷단에 실제 LLM을 붙여 실행할 에이전트 설계안을 클로드 스킬에 맞게 변환하는 작업이 필요하다. 클로드를 쓰면서 쉽게 쉽게 스킬로 불러오려면 명칭도, 룰도, 다시 설정할 필요성이 있다.

##### 에이전트 이름 재정의

| 원래 ID | 원래 이름              | 제안 스킬 이름        | MEMO                                              |
| ----- | ------------------ | --------------- | ------------------------------------------------- |
| AG-OR | Orchestrator       | **rc**          | 모든 입력의 첫 관문, 라우팅이 핵심 동작<br>→ 이젠 이게 상위 스킬 역할을 하게된다 |
| AG-BD | Book Discriminator | **rc_register** | 책을 식별하고 Vault에 '등록'하는 행위가 본질                      |
| AG-TE | Text Extract       | **rc_save**     | 책 콘텐츠, 하이라이트 등을 저장하는 행위가 본질                       |
| AG-DG | Dialogue           | **rc_talk**     | 대화하고 추천하는 에이전트, 가장 단순한 이름이 적합                     |
| AG-IS | Insight Synthesis  | **rc_ontology** | 유저의 지식 온톨로지를 만들고 유지하는 엔진                          |
