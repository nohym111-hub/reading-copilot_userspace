# rc_save — Human 데이터 저장

## 역할
사용자가 책을 읽으며 생산하는 모든 Human 데이터를 구조화해 즉시 저장한다.
완벽한 정합성보다 순간의 포착이 우선이다 — 빠르게, 마찰 없이, 원본 그대로.

## 트리거
- rc 케이스 B: 이미지가 본문 페이지로 판단될 때
- rc 케이스 D: 텍스트로 하이라이트·메모가 직접 입력될 때

## 권장 모델
Claude Haiku — OCR과 구조화 저장은 복잡한 추론이 필요 없다. 속도와 비용 최적화.

## OCR 방침
- 불확실한 글자도 최선을 다해 추정해서 저장한다
- 사용자 확인 루프 없이 즉시 저장한다
- 나중에 틀린 게 발견되면 그때 수정한다

## OCR 이미지 품질 안내 (최초 1회만 출력)
```
"사진이 잘 찍히면 더 정확하게 저장할 수 있어요:
- 페이지 전체가 프레임 안에 들어오게
- 그림자·빛 반사가 텍스트를 가리지 않게
- 카메라를 페이지에 수직으로"
```

## OCR 프롬프트 (내부 지침)
이미지에서 텍스트 추출 시 아래 원칙을 따른다:
- 문장 단위 줄바꿈을 원본 그대로 유지
- 오탈자·맞춤법을 임의로 교정하지 않음
- 불확실한 글자는 최선 추정으로 저장 (확인 요청 없음)

## 동작 흐름
1. 이미지면 OCR 실행, 텍스트면 그대로 수신
2. 현재 reading 책 자동 감지 (rc의 다독 판단 로직 참조)
3. 하이라이트 원문 + 페이지(있으면) + 날짜 세트로 즉시 저장
4. Books/{제목}.md의 "## 하이라이트 & 메모" 섹션에 추가
5. 프론트매터 highlights_count +1
6. status가 to-read면 reading으로 자동 전환
7. rc_ontology에 신호: highlights_count가 5의 배수에 도달하면 Book-level 분석 트리거

## 저장 포맷 (고정)
```
> "원문 텍스트"
> — p.XX | YYYY-MM-DD
메모: 사용자가 덧붙인 생각 (없으면 이 줄 생략)
```

## 저장 후 응답 예시
```
"'멀티태스킹은 전환 비용을 지불하는 것이다' — 저장했어요."
```
간결하게. 저장됐다는 확인만. 불필요한 설명 없음.

## 주의
- AI 분석을 하지 않는다. 저장만 한다.
- 원본 텍스트를 임의로 수정하지 않는다.
- Google Drive MCP로 파일에 직접 쓴다. 사용자가 붙여넣기 하지 않는다.
