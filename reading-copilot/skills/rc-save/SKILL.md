# rc-save v3

## 역할 분리 (핵심 원칙)

| 담당 | 역할 |
|---|---|
| **Claude** | 이미지 OCR + 마킹된 텍스트 감지 → JSON 출력 |
| **rc_save.py** | JSON → Markdown 변환, 페이지 경계 병합, Obsidian 저장, 검증, 임시파일 삭제 |

Claude는 Markdown을 직접 생성하지 않는다.
포맷 결정권은 코드에 있다 — 세션이 길어져도 출력 형식이 변하지 않는다.

---

## 트리거
- 책 본문 페이지 이미지가 첨부될 때
- "저장해줘", "하이라이트 추가", "이 부분 기록해줘"
- Dispatch로 이미지 + 책 제목이 전달될 때

## 사전 요건 (Configuration)

CLAUDE.md의 `## ⚙ Configuration` 블록에서 읽는다:
```yaml
script_path: /path/to/skills/rc-save/rc_save.py

```
Configuration이 없으면 → "rc-setup을 먼저 실행해주세요" 후 중단.

## 실행 흐름

### STEP 1 — 책 특정
우선순위:
1. 같은 대화에서 rc_register가 실행됐으면 → 해당 경로 그대로 사용
2. 사용자가 책 제목을 언급했으면 → `--book "제목"` 인자로 전달
3. 둘 다 없으면 → "어떤 책 페이지인가요?" 한 줄 질문 후 대기

### STEP 2 — Claude OCR (JSON 출력만)

각 이미지에서 아래 항목을 추출해 **JSON만 출력**한다.
설명, 마크다운, 전처리 텍스트 일절 출력 금지.

#### OCR 원칙
- 보이는 텍스트를 **글자 그대로** 옮긴다
- 오탈자·비문·어색한 표현 — 수정하지 않는다
- 불확실한 글자 → 최선을 다해 추정, 건너뛰지 않는다

#### 마킹 감지 원칙
아래 시각적 특징을 가진 텍스트를 `highlighted`로 분류한다:
- 텍스트 아래에 선이 그어진 경우 (밑줄)
- 텍스트 배경에 색이 칠해진 경우 (형광펜)
- 텍스트 옆에 세로 선이 그어진 경우 (여백 바)

판단이 불명확하면 → 마킹으로 처리한다 (누락보다 과포함이 낫다)

#### 페이지 경계에서 잘린 하이라이트 처리
마킹된 텍스트가 페이지 하단에서 잘릴 때:
- `highlighted`에 잘린 텍스트 그대로 넣는다 (붙이거나 추정하지 않는다)
- `after`는 빈 문자열로 둔다
- 다음 페이지 첫 줄이 이어지면 → 다음 페이지 quote의 `highlighted` 첫 부분에 나머지를 넣는다
- rc_save.py의 `should_merge()`가 두 quote를 자동으로 하나로 합친다

병합 판단은 코드에 위임한다 — Claude가 직접 붙이지 않는다.

#### 문맥 추출 원칙
- `before`: 하이라이트 **직전 1문장**만. 없으면 빈 문자열.
- `after`: 하이라이트 **직후 1문장**만. 없으면 빈 문자열.
- 문장이 불명확하면 가장 가까운 완전한 문장 1개를 선택한다

#### 메모 감지 원칙
- 인쇄된 본문과 다른 필체(손글씨)를 메모로 분류
- 메모가 없으면 빈 문자열

#### 출력 JSON 형식 (이 형식 외 다른 텍스트 출력 금지)

```json
[
  {
    "page": 84,
    "quotes": [
      {
        "highlighted": "마킹된 텍스트 원문 — 글자 그대로",
        "before": "하이라이트 직전 1문장",
        "after": "하이라이트 직후 1문장",
        "memo": "여백 손글씨 메모 (없으면 빈 문자열)"
      }
    ]
  }
]
```

규칙:
- 이미지 1장 = 배열 원소 1개
- 이미지 여러 장 = 페이지 순서대로 배열
- 한 페이지에 마킹 없음 → `"quotes": []`
- 한 페이지에 마킹 여러 개 → `quotes` 배열에 순서대로 나열
- 페이지 번호를 찾을 수 없으면 → `"page": null`

### STEP 3 — 스크립트 실행

출력한 JSON을 rc_save.py에 파이프로 전달한다:

```bash
echo '<STEP 2에서 출력한 JSON>' | python {script_path} \
  --book "{책 제목}" \
  --vault "{vault_root}"
```

또는 임시 JSON 파일을 경유하는 방식 (저장 후 자동 삭제):

```bash
python {script_path} \
  --book "{책 제목}" \
  --vault "{vault_root}" \
  --json-file "/tmp/ocr_result.json"
```

`--json-file`을 사용하면 저장 성공 후 해당 파일이 자동으로 삭제된다.

실행 중 중간 상태를 사용자에게 출력하지 않는다.

### STEP 4 — 완료 메시지

스크립트 stdout의 마지막 줄을 그대로 출력한다:
```
✓ N장 처리 완료 | 하이라이트 M개 저장 | 책제목
```

검증 실패(exit code 2)가 있으면 stderr 경고도 함께 출력한다.

## 저장 포맷 (rc_save.py가 생성 — Claude가 결정하지 않음)

```
> "before **highlighted** after"
> — p.XX | YYYY-MM-DD
메모: 여백 손글씨 (없으면 이 줄 생략)
```

페이지 경계 병합 시:
```
> "before **highlighted_p1 highlighted_p2** after"
> — p.XX–YY | YYYY-MM-DD
```

실제 예시:
```
> "하지만 '제대로' 치기 위해 의식적으로 노력하면 안 된다는 점을 명심해야 한다. **당신의 스윙에서 어떤 부분이 변하고 있는지를 계속 지켜보기만 하면 된다. 다른 사람의 스트로크를 관찰하듯이 한 걸음 떨어져서 관심을 가지고 지켜보자. 단지 보기만 해도 자연스러운 과정을 통해 스트로크가 바뀔 것이다.** 사실 이는 믿기 어려운 말처럼 들릴 수도 있다."
> — p.85–86 | 2026-06-03
```

## rc_save.py 병합 로직 (참고)

`should_merge(prev, next)` 판단 기준:
- 이전 quote가 문장 종결부호(`.?!」"'。`)로 끝나면 → 병합 안 함
- 이전 quote가 한국어 종결 패턴(`[가-힣]다`, `[가-힣][요죠나군]`)으로 끝나면 → 병합 안 함
- 다음 quote가 영문 대문자로 시작하면 → 병합 안 함
- 위 조건 모두 해당 없으면 → 병합

공백 처리:
- 영어 하이픈 분절(`exam-` + `ple`) → 하이픈 제거 후 붙임
- 한글-한글 경계 → 공백 없이 붙임 (`시` + `리즈` = `시리즈`)
- 그 외 → 공백 한 칸으로 이음

## 오류 처리

| 상황 | 출력 |
|---|---|
| Configuration 없음 | "rc-setup을 먼저 실행해주세요." |

| 검증 실패 | 실패 항목 + "재시도하거나 수동 확인이 필요해요." |
