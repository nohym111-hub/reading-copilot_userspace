# rc-save v4

## 역할 분리 (핵심 원칙)

| 담당 | 역할 |
|---|---|
| **Claude** | 이미지 OCR + 마킹/카드 감지 → JSON 출력, 이미지 간 중복 사전 제거 |
| **rc_save.py** | JSON → Markdown 변환, 페이지 경계 병합, 기저장 중복 제거, Obsidian 저장, 검증 |

Claude는 Markdown을 직접 생성하지 않는다.
포맷 결정권은 코드에 있다 — 세션이 길어져도 출력 형식이 변하지 않는다.

---

## 트리거
- 책 본문 페이지 이미지가 첨부될 때
- 독서 앱 하이라이트 목록 스크린샷이 첨부될 때 (밀리의 서재 등)
- "저장해줘", "하이라이트 추가", "이 부분 기록해줘"
- **텍스트로 감상·리뷰·생각을 직접 입력할 때** → Mode C
- Dispatch로 이미지 + 책 제목이 전달될 때

---

## 사전 요건 (Configuration)

CLAUDE.md의 `## ⚙ Configuration` 블록에서 읽는다:
```yaml
script_path: /path/to/skills/rc-save/rc_save.py
vault_root:  /path/to/MyVault
books_dir:   Books
```
Configuration이 없으면 → "rc-setup을 먼저 실행해주세요" 후 중단.

## 실행 흐름

### STEP 1 — 책 특정

우선순위:
1. 같은 대화에서 rc_register가 실행됐으면 → 해당 경로 그대로 사용
2. **이미지 중 책 표지가 있으면** (첫 번째 이미지가 표지인 경우가 많음) → 표지에서 제목 추출
3. 사용자가 책 제목을 언급했으면 → `--book "제목"` 인자로 전달
4. 모두 없으면 → "어떤 책 페이지인가요?" 한 줄 질문 후 대기

---

### STEP 1.5 — 입력 유형 감지

| 유형 | 판단 기준 |
|---|---|
| **Mode A: 책 페이지** | 이미지 — 책 본문 레이아웃, 페이지 번호, 여백, 마킹(형광펜·밑줄·여백바) |
| **Mode B: 앱 스크린샷** | 이미지 — iOS/Android 상태바, 카드 리스트 UI, `"` 아이콘 + 인용문 + 날짜 반복 |
| **Mode C: 텍스트 감상** | 이미지 없음 + 사용자가 직접 쓴 감상·리뷰·생각 텍스트 |

같은 배치에 두 유형이 섞이면 → 이미지별로 유형을 판단해 각각 해당 모드로 처리.
Mode C는 STEP 2 OCR을 건너뛰고 → **STEP 2C**로 직접 이동한다.

---

### STEP 2C — Mode C: 텍스트 감상 직접 저장 (OCR 없음)

이미지가 없고 사용자가 감상·리뷰 텍스트를 직접 입력한 경우.

1. 책 파일(`{vault_root}/{books_dir}/{제목}.md`)의 `## 한줄 인상 (Human)` 섹션을 찾는다.
2. 다음 포맷으로 텍스트를 추가한다 (filesystem Write/Edit 도구 사용):

```
> **YYYY-MM-DD**
>
> 첫 번째 단락
>
> 두 번째 단락
> - 목록 항목은 `> -` 로
>
> 마지막 단락
```

규칙:
- 날짜는 오늘 날짜 (CLAUDE.md `currentDate` 참조)
- 사용자 텍스트의 단락 구분은 빈 `>` 줄로 유지한다
- 목록(`-`)은 `> -` 형태로 변환한다
- 텍스트 내용을 요약하거나 수정하지 않는다 — **글자 그대로** 저장
- rc_save.py를 호출하지 않는다 (JSON 파이프라인 불필요)

4. 저장 후 완료 메시지:
```
✓ 감상 저장 완료 | 한줄 인상 섹션 | {제목}
```

---

### STEP 2 — Claude OCR (JSON 출력만)

설명, 마크다운, 전처리 텍스트 일절 출력 금지. **JSON만 출력**.

---

#### Mode A: 책 페이지 (physical book)

##### OCR 원칙
- 보이는 텍스트를 **글자 그대로** 옮긴다
- 오탈자·비문·어색한 표현 — 수정하지 않는다
- 불확실한 글자 → 최선을 다해 추정, 건너뛰지 않는다

##### 마킹 감지 원칙
아래 시각적 특징을 가진 텍스트를 `highlighted`로 분류한다:
- 텍스트 아래에 선이 그어진 경우 (밑줄)
- 텍스트 배경에 색이 칠해진 경우 (형광펜)
- 텍스트 옆에 세로 선이 그어진 경우 (여백 바)

판단이 불명확하면 → 마킹으로 처리한다 (누락보다 과포함이 낫다)

##### 페이지 경계에서 잘린 하이라이트 처리
마킹된 텍스트가 페이지 하단에서 잘릴 때:
- `highlighted`에 잘린 텍스트 그대로 넣는다 (붙이거나 추정하지 않는다)
- `after`는 빈 문자열로 둔다
- 다음 페이지 첫 줄이 이어지면 → 다음 페이지 quote의 `highlighted` 첫 부분에 나머지를 넣는다
- rc_save.py의 `should_merge()`가 두 quote를 자동으로 하나로 합친다

병합 판단은 코드에 위임한다 — Claude가 직접 붙이지 않는다.

##### 문맥 추출 원칙
- `before`: 하이라이트 **직전 1문장**만. 없으면 빈 문자열.
- `after`: 하이라이트 **직후 1문장**만. 없으면 빈 문자열.

##### 메모 감지 원칙
- 인쇄된 본문과 다른 필체(손글씨)를 메모로 분류
- 메모가 없으면 빈 문자열

##### 출력 JSON 형식 (Mode A)

```json
[
  {
    "page": 84,
    "quotes": [
      {
        "highlighted": "마킹된 텍스트 원문 — 글자 그대로",
        "before": "하이라이트 직전 1문장",
        "after": "하이라이트 직후 1문장",
        "memo": "여백 손글씨 메모 (없으면 빈 문자열)"
      }
    ]
  }
]
```

규칙:
- 이미지 1장 = 배열 원소 1개
- 한 페이지에 마킹 없음 → `"quotes": []`
- 한 페이지에 마킹 여러 개 → `quotes` 배열에 순서대로 나열
- 페이지 번호를 찾을 수 없으면 → `"page": null`

---

#### Mode B: 앱 스크린샷 (e-book, 밀리의 서재 등)

##### 카드 추출 원칙
각 하이라이트 카드에서 아래 3가지를 추출한다:
1. **`highlighted`**: `"` 아이콘 이후 인용 텍스트 원문 — 글자 그대로
2. **`memo`**: 회색 말풍선(사용자 코멘트) 텍스트. 없으면 빈 문자열
3. **`timestamp`**: 카드 하단 날짜+시각 (예: `"2025.09.09 08:56"`)

`before`, `after`는 항상 빈 문자열 — 앱이 문맥을 제공하지 않으므로.
`page`는 항상 `null` — 전자책은 페이지 번호가 표시되지 않으므로.

##### 연속 스크린샷 중복 제거 (핵심)

연속 스크린샷은 스크롤 경계에서 동일 카드가 겹친다.
**처리 규칙 (Claude가 이미지 간 사전 제거):**
1. 모든 이미지를 순서대로 훑으며 추출한 항목의 **`timestamp`를 지문**으로 관리
   - timestamp가 없으면 → `highlighted` 앞 40자를 지문으로 사용
2. 이미 추출한 지문이 후속 이미지에서 나타나면 → JSON에서 제외
3. 제외된 항목은 JSON에 포함하지 않는다 — 최종 JSON에는 각 항목이 1회만 등장

##### 출력 JSON 형식 (Mode B)

```json
[
  {
    "page": null,
    "quotes": [
      {
        "highlighted": "인용 텍스트 원문",
        "before": "",
        "after": "",
        "memo": "사용자 메모 (없으면 빈 문자열)",
        "timestamp": "2025.09.09 08:56"
      }
    ]
  }
]
```

규칙:
- 이미지 1장 = 배열 원소 1개 (책 표지는 제외)
- 카드가 없는 스크린샷 → `"quotes": []`
- 중복 제거 후 새로운 항목이 없어도 → `"quotes": []` (빈 배열, 배열 원소는 유지)

---

### STEP 3 — 스크립트 실행

출력한 JSON을 rc_save.py에 전달한다:

```bash
python {script_path} \
  --book "{책 제목}" \
  --vault "{vault_root}" \
  --json-file "/tmp/ocr_result.json"
```

`--json-file`을 사용하면 저장 성공 후 해당 파일이 자동으로 삭제된다.

실행 중 중간 상태를 사용자에게 출력하지 않는다.

---

### STEP 4 — 완료 메시지

스크립트 stdout의 마지막 줄을 그대로 출력한다:
```
✓ N장 처리 완료 | 하이라이트 M개 저장 (중복 K개 건너뜀) | 책제목
```

검증 실패(exit code 2)가 있으면 stderr 경고도 함께 출력한다.

## 저장 포맷 (rc_save.py가 생성 — Claude가 결정하지 않음)

**Mode A (페이지 번호 있음):**
```
> "before **highlighted** after"
> — p.XX | YYYY-MM-DD
메모: 여백 손글씨 (없으면 이 줄 생략)
```

**Mode B (앱 스크린샷, timestamp 있음):**
```
> "**highlighted**"
> — 2025.09.09 08:56 | YYYY-MM-DD
메모: 사용자 코멘트 (없으면 이 줄 생략)
```

**페이지 번호도 timestamp도 없을 때:**
```
> "**highlighted**"
> — YYYY-MM-DD
```

실제 예시 (Mode A):
```
> "하지만 '제대로' 치기 위해 의식적으로 노력하면 안 된다는 점을 명심해야 한다. **당신의 스윙에서 어떤 부분이 변하고 있는지를 계속 지켜보기만 하면 된다. 다른 사람의 스트로크를 관찰하듯이 한 걸음 떨어져서 관심을 가지고 지켜보자. 단지 보기만 해도 자연스러운 과정을 통해 스트로크가 바뀔 것이다.** 사실 이는 믿기 어려운 말처럼 들릴 수도 있다."
> — p.85–86 | 2026-06-03
```

실제 예시 (Mode B):
```
> "**아마 씻다르타가 근사한 남정네라서, 그의 눈길이 여인네들을 반하게 만들어, 그 때문에 그에게 행운이 찾아오는 것인지도 모르죠.**"
> — 2025.09.09 08:56 | 2026-06-21
메모: 시온이 ✨✨✨
```

## rc_save.py 중복 제거 로직 (참고)

### 기저장 항목 중복 제거
저장 전에 노트 파일에서 기존 지문(fingerprint)을 추출:
- timestamp 패턴 (`> — YYYY.MM.DD HH:MM`) → `ts:YYYY.MM.DD HH:MM`
- `**highlighted**` 텍스트 앞 40자

새 항목의 지문이 기존 지문과 일치하면 → 건너뜀 (stderr에 건너뜀 수 출력).

### 페이지 경계 병합 (Mode A 전용)
`should_merge(prev, next)` 판단 기준:
- 이전 quote가 문장 종결부호(`.?!」"'。`)로 끝나면 → 병합 안 함
- 이전 quote가 한국어 종결 패턴(`[가-힣]다`, `[가-힣][요죠나군]`)으로 끝나면 → 병합 안 함
- 다음 quote가 영문 대문자로 시작하면 → 병합 안 함
- 위 조건 모두 해당 없으면 → 병합

Mode B 항목은 완결된 문장이므로 자연스럽게 병합 조건에 해당하지 않는다.

## 오류 처리

| 상황 | 출력 |
