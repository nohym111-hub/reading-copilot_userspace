## 2. PRD 서비스 취지 (채점 기준)
PRD가 온톨로지에 부여한 역할은 명확하다 — **"모든 데이터를 분석해 다른 스킬에 컨텍스트를 공급하는 두뇌"**(§2, §4.2). 핵심 원칙:
- rc_talk에 대화·추천 컨텍스트 공급 / rc에 라우팅 힌트
- **공백 탐지**(profile.md "탐색 공백")
- **동적·무설정 생성**("초반 세팅 없이 완전히 동적으로", §3.6)
- **Ambient Persistence & 확장성**(100권+ 캐시, §3.1)
- Human+AI 분리

## 4. 해석
- **V0가 PRD에 가장 부합한다(28/35).** 당연하다 — PRD §3.2가 곧 V0의 설계도다. 무설정·창발·공백탐지·캐시 구조가 PRD의 "Intent-First / Seamless / Ambient" 철학과 1:1로 맞는다. 추천과 공백 탐지(③②)는 PRD가 온톨로지에 기대한 1차 기능인데 V0가 가장 직접적으로 수행한다.
- **V1은 "미래의 V0"다.** 단권 분석을 넘어 *책 사이의 깊은 연결*과 *심층 대화 컨텍스트*에서 압도적이고, 100권+ 확장성도 최고다. 다만 엔티티 정제에 비용이 들어 "무설정 창발"(③) 원칙과 충돌한다 → 초기 단계엔 과하다.
- **V2는 "이 사용자에게 가장 잘 맞는다"(⑦ 5점)** 는 역설. 사용자는 명백히 목표지향적 독자(제안·커리어·Reading Copilot 제작)라 OKR이 동기를 강하게 건드린다. 그러나 top-down 목표 설정이 PRD의 Intent-First/창발 원칙(③)과 정면 충돌하고, [[소금. 산. 지방. 불]] 같은 무목적 독서를 "정렬 실패"로 깎는 부작용이 있다.

## 5. 권고: V0를 중핵으로, V1·V2를 레이어로 흡수
PRD 취지 1순위는 **V0**다. 다만 단독으로 두지 말고 단계적으로 V1·V2 요소를 흡수하는 하이브리드를 권한다.

```
[ V0 관심사 그래프 ]  ← 항상 가동되는 중핵 (PRD 네이티브)
        │
        ├─(개념이 3권+ 등장 시)→ [ V1 엔티티 페이지 자동 승격 ]  : 확장성·심층대화 보강
        │
        └─(rc_talk 프로액티브 시)→ [ V2 목표 렌즈 ]              : 동기·의도적 공백 메우기
```

- **중핵 V0**: profile.md + themes.md를 표준 캐시로 계속 사용.
- **V1 흡수 트리거**: 한 개념이 책 3권 이상에서 등장하면(현재 [[정보 네트워크]]·[[몰입]]·[[질서의 허구성]] 등 충족) 그 개념을 엔티티 페이지로 승격해 양방향 링크·근거(claim)를 붙인다. 확장성 문제를 "필요할 때만" 해결.
- **V2 흡수 방식**: 목표를 강제 설정하지 않고, rc_talk가 *추론한* 목표를 "이런 방향을 향하시는 것 같아요"로 제안만 한다. 공백 탐지를 "목표 대비 미달 KR"로 표현하면 추천 설득력이 커진다(예: O1 인풋 과잉→"덜 읽고 더 말하기" 제안).

## 6. 향후 발전 방향 제언
1. **공백→행동 루프 (최우선).** 세 버전이 공통으로 짚은 최대 공백은 **'아웃풋 부재'** — Books의 "대화 기록" 섹션이 전부 비어 있다(V2 KR1.3·KR3.2 = 0.1~0.3). 온톨로지의 다음 목표는 더 많은 분석이 아니라 **rc_talk를 돌려 사용자의 재생산(대화·요약)을 유도**하는 것. 이것이 PRD의 rc_talk 프로액티브 모드와 정확히 맞물린다.
2. **개념 승격 자동화.** "3권 등장 → 엔티티 페이지 생성" 규칙을 rc-ontology Cross-book 레이어에 추가. V1의 장점을 무설정으로 얻는다.
3. **양방향 링크 무결성.** 불문율(Books 미수정) 하에서는 책↔테마 연결이 한쪽(Ontology)에만 적힌다. `themes.md` 부록 표를 **연결의 단일 진실원본**으로 삼고, rc_talk가 책 컨텍스트 로드 시 이 표를 역참조하는 규약을 둔다. (PRD가 ai_connected_books를 책 프론트매터에 두던 설계와의 트레이드오프를 문서로 명시)
4. **'무목적 독서' 트랙 보존.** 어떤 모델을 쓰든 [[소금. 산. 지방. 불]]·[[Gattaca]]처럼 목표·연결이 약한 즐거움 독서를 깎지 않는 별도 칸을 유지. 독서 동기의 다양성을 지키는 안전장치.
5. **스텁 정책 명문화.** 듀얼 브레인·RAFA·참교육·파이 이야기처럼 데이터 없는 노트엔 노드를 생성하지 않는다(환각 방지)는 원칙을 rc-ontology SKILL.md에 추가.
6. **시간축 활용.** profile.md의 시간축(5월 자아탐구→6월 기술집중→6/21 몰입회복)은 V0만 포착한다. rc_talk Remind 모드가 이 궤적을 읽어 "지난달의 질문으로 돌아가 볼까요?"식 회고를 제안하도록 연결.

## 7. 한 줄 결론
> **PRD 취지엔 V0(Ontology/)가 가장 부합한다. V1은 규모가 커질 때의 정답, V2는 이 사용자의 동기에 맞는 렌즈다. → V0를 중핵으로 두고 V1(개념 승격)·V2(목표 렌즈)를 조건부 레이어로 흡수하는 것이 최선이며, 다음 한 걸음은 "분석 추가"가 아니라 "rc_talk로 아웃풋(대화·재생산)을 끌어내 비어 있는 대화 기록을 채우는 것"이다.**
